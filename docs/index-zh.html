01 · 功能特性

为 Resolve 工作流
量身打造

每个功能都精准解决 DaVinci Resolve Free 在 Linux 平台上的实际痛点。不多不少，恰到好处。

01

mpv 视频预览

mpv 视频输出通过 X11 子窗口直接嵌入 GUI 界面。 完整的播放控制，支持逐帧精确步进——跳转、拖拽、 单帧微调，一气呵成。

02

键盘优先操作

播放过程中按 I 标记入点、O 标记出点。 每次按 O 自动捕获片段。批量剪辑， 双手无需离开键盘。

03

DNxHD / DNxHR 导出

仅转码你需要的片段。ffmpeg 支持 LB、SQ、HQ、HQX 四种 DNx 编码配置。输出 .mov 文件，DaVinci Resolve 原生支持，直接导入。

04

片段管理

从同一源文件中截取多个出入点范围。侧边栏中查看 每个片段的起止时间和时长。导出前可随时移除 不需要的片段。

05

拖放打开

将视频文件拖入预览区域即可打开，或使用 Ctrl+O 调用原生文件对话框。 支持 mp4、mov、mkv、avi、webm、mts 等主流格式。

06

智能导出命名

输出文件自动以源文件名和片段时间戳命名。 若文件已存在，导出前会弹出提示， 避免意外覆盖。

02 · 系统架构

三个进程，
一条无缝管线

dnclip 通过统一的 Rust 图形界面，将三个久经考验的 工具串联起来——各司其职，协同工作。

图形界面层
egui + eframe
→ ↓
播放引擎
mpv
→ ↓
编码器
ffmpeg

egui 渲染界面

Rust 语言的即时模式 GUI。时间轴滑块、走带控件、 片段列表和导出面板——全部以原生速度渲染， 资源占用极低。

mpv 解码与渲染

作为子进程启动，通过 X11 窗口重定父级嵌入 egui 窗口。通过 Unix 套接字使用 JSON IPC 进行控制—— 播放、暂停、跳转、逐帧、查询属性。

ffmpeg 编码输出

每次导出时调用，精确传递 -ss 和 -t 参数。编码为 DNxHD/DNxHR 视频 加 PCM 音频，封装为 .mov 容器——Resolve 即开即用。

03 · 技术规格

技术参数

开发语言 Rust（2021 版）—— 6 个源文件，共 1,411 行代码
GUI 框架 egui 0.34 + eframe 运行时
视频播放 mpv，通过 X11 子窗口嵌入 + JSON IPC 通信
导出编码 ffmpeg CLI —— DNxHD/DNxHR 配置：LB、SQ、HQ、HQX
输出格式 QuickTime .mov —— DNx 视频 + PCM 16-bit 音频
窗口系统 X11（x11-dl）实现原生子窗口嵌入
序列化 serde + serde_json 处理 mpv IPC 协议
文件对话框 rfd（原生）+ egui-file-dialog（GUI 内嵌）
打包方式 cargo-deb，含 .desktop 启动项与应用图标
输入格式 mp4 mov mkv avi webm mts m2ts ts flv wmv
开源协议 MIT
04 · 快速开始

安装指南

系统依赖
# Ubuntu / Debian sudo apt install mpv ffmpeg libx11-dev
使用 Cargo 安装
$ cargo install dnclip
或从源码编译
$ git clone https://github.com/yinguobing/dnclip.git $ cd dnclip $ cargo build --release $ ./target/release/dnclip
